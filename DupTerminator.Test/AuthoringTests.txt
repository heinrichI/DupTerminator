==========================================================================
Visual Studio Team System: общие сведения о разработке и выполнении тестов
==========================================================================

В общих сведениях описываются возможности разработки и выполнения тестов в 
средствах Visual Studio Team System и Visual Studio Team Edition for 
Software Testers.

Открытие тестов
---------------
Чтобы открыть тест, откройте проект тестирования или файл метаданных теста 
(файл с расширением VSMDI), который содержит определение этого теста. 
Проекты и файлы метаданных тестов доступны в обозревателе решений.

Просмотр тестов
---------------
Чтобы узнать, какие тесты доступны, откройте окно «Представление теста». Если 
установлено средство Team Edition for Software Testers, можно также просмотреть 
доступные тесты в окне «Редактор списка тестов».

Чтобы открыть окно «Представление теста», откройте меню «Тест», установите 
указатель мыши на подменю «Окна» и выберите команду «Представление теста». 
Чтобы открыть окно «Редактор списка тестов» (если установлено средство Team 
Edition for Software Testers), откройте меню «Тест», установите указатель на 
команду «Окна» и выберите команду «Редактор списка тестов».

Выполнение тестов
-----------------
Тесты можно выполнять в окнах «Представление теста» и «Редактор списка тестов».
Сведения о доступе к этим окнам см. в разделе «Просмотр тестов». Чтобы 
выполнить один или несколько тестов из отображаемых в окне «Представление 
теста», сначала выберите тесты в этом окне; чтобы выбрать несколько тестов, 
щелкните нужные тесты, удерживая нажатой клавишу Shift или CTRL. Затем нажмите 
кнопку «Выполнить тесты» в панели инструментов окна «Представление теста».

Если установлено средство Visual Studio Team Edition for Software Testers, 
выполнение тестов также можно осуществлять в окне «Редактор списка тестов». 
Чтобы выполнить тесты в окне «Редактор списка тестов», установите флажки рядом 
с тестами, которые нужно выполнить. Затем нажмите кнопку «Выполнить тесты» в 
панели инструментов окна «Представление теста».

Просмотр результатов тестов
---------------------------
Результаты выполнения отдельного теста или серии тестов отображаются в окне 
«Результаты теста». Результаты выполнения каждого из тестов показаны на 
отдельной строке, что позволяет определить состояние соответствующего теста. В 
верхней части этого окна находится встроенный индикатор состояния, в котором 
приводятся сводные сведения о выполнении тестов в целом.

Чтобы просмотреть подробные сведения о результатах выполнения конкретного 
теста, следует дважды щелкнуть его в окне «Результаты теста». Будет открыто 
окно, содержащее более подробные сведения о результатах выполнения отдельного 
теста, например сообщения о конкретных ошибках, созданные в ходе этого теста.

Изменение хода выполнения тестов
--------------------------------
Каждый раз при выполнении одного или нескольких тестов применяется коллекция 
параметров, определяющих ход выполнения этих тестов. Эти параметры содержатся в 
файле «конфигурации тестирования».

Ниже приводится неполный список изменений, доступных с использованием файла 
конфигурации тестирования:

 - Изменение схемы именования для каждого выполнения тестов.
 - Изменение контроллера тестов, на котором выполняется тестирование, что 
   обеспечивает возможность удаленного выполнения тестов.
 - Сбор данных по покрытию кода, позволяющий понять, какие строки кода 
   покрываются тестами.
 - Включение и отключение развертывания тестов. 
 - Определение дополнительных файлов для развертывания перед выполнением тестов.
 - Изменение хост-узла на ASP.NET для выполнения модульных тестов ASP.NET.
 - Изменение хост-узла на узел тестирования интеллектуальных устройств для 
   выполнения модульного теста интеллектуального устройства.
 - Настройка различных свойств для агентов тестирования, на которых выполняются 
   тесты.
 - Выполнение пользовательских сценариев в начале и конце каждого тестирования 
   для точной настройки среды тестирования в соответствии с потребностями.
 - Установка ограничений по времени для тестов и их выполнения.
 - Настройка набора обозревателей и количества повторов для веб-тестов при 
   тестировании.

По умолчанию файл конфигурации тестирования создается при создании нового 
проекта тестирования. Чтобы внести изменения в этот файл, следует дважды 
щелкнуть его в обозревателе решений и изменить нужные параметры. (Файлы 
конфигурации тестирования имеют расширение TESTRUNCONFIG.)

Решение может содержать несколько файлов конфигурации тестирования. Для 
определения параметров, используемых при текущем тестировании, используется 
только один из этих файлов, который имеет статус «Активно». Чтобы выбрать 
активную конфигурацию тестирования, следует в меню «Тест» выбрать команду 
«Выбрать активную конфигурацию тестирования».

Типы тестов
-----------
При работе со средством Visual Studio Team Edition for Software Testers можно 
создать ряд типов тестов:

Модульный тест: этот тест позволяет создать программный тест на языке C++, 
Visual C# или Visual Basic, в котором будет выполняться исходный код. В 
модульном тесте выполняется вызов методов класса, передаются подходящие 
параметры и проверяется соответствие возвращаемого значения ожидаемому.
Существует три специальных варианта модульных тестов:
 - Управляемые данными модульные тесты создаются в том случае, если тест модуля 
   должен многократно вызываться для каждой строки источника данных. Данные 
   соответствующей строки используются в этом тесте как входные данные.
 - В модульных тестах ASP.NET выполняется код веб-приложений ASP.NET.
 - Модульные тесты интеллектуального устройства представляют собой модульные 
   тесты, которые развертываются на интеллектуальном устройстве или в 
   эмуляторе, а затем выполняются хост-узлом тестирования этого 
   интеллектуального устройства. 

Веб-тест: веб-тесты состоят из упорядоченной серии запросов HTTP, записанных в 
сеансе обозревателя с использованием Microsoft Internet Explorer. В тесте могут 
выводиться подробные сведения о запрашиваемых узлах и страницах. Например, этот 
тест позволяет выявить, содержит ли конкретная страница указанную строку.

Нагрузочный тест: в этом тесте могут быть инкапсулированы автоматические тесты, 
например тесты модуля, веб-тесты и универсальные тесты, которые затем будут 
выполнены одновременно с применением виртуальных пользователей. В результате 
выполнения этих тестов с нагрузкой можно получить данные, в том числе данные о 
производительности, представленные в виде таблиц и диаграмм.

Универсальный тест: этот тест представляет собой существующую программу с 
оболочкой для функционирования в качестве теста в Visual Studio. Ниже 
приводятся примеры тестов или программ, которые можно преобразовать в 
универсальные тесты.
 - Существующий тест, в котором при помощи кодов завершения процесса 
   указывается, пройден тест или нет. Значение 0 указывает, что тест пройден, 
   любое другое значение — что тест не пройден.
 - Общая программа, позволяющая получить конкретную функциональность в тестовом 
   сценарии.
 - Тест или программа, результаты выполнения которого выводятся в подробном 
   виде при помощи особого XML-файла («сводного файла результатов»).

Ручной тест: ручной тест используется в том случае, если задачи тестирования 
должны быть выполнены специалистом, осуществляющим тестирование. 
Противоположностью ручного теста является автоматический сценарий.

Упорядоченный тест: упорядоченное тестирование позволят выполнить набор тестов 
в указанном порядке.
